The Two Roll Types

🪞 Mirror Roll

What: Opens an opposite-side spread on the same expiration day. Original (threatened) position stays open.

How:

  • Threatened PUT spread → opens CALL spread
  • Threatened CALL spread → opens PUT spread
  • New short strike near current price (ATM)
  • Same width as original

When it triggers:

  • Expiration day only
  • Within configured UTC window (default 18:00–20:00 = 11am–1pm PST)
  • Severity ≥ mirror_trigger_severity (default warning)

Goal: collect premium on the opposite side to offset the threatened side

➡️ Forward Roll

What: Closes current spread, opens same-side spread at a further DTE.

How:

  • PUT stays PUT, CALL stays CALL
  • New strikes pushed further OTM (default max(1%, current_distance%))
  • Same width as original
  • DTE range: 1–5 days forward

When it triggers:

  • Any trading day (skips weekends)
  • Severity ≥ forward_trigger_severity (default watch)

Goal: extend the position with reduced breach risk and additional time

Breach Severity Levels

Every open spread is classified on a 5-level scale based on price distance from the short strike:

safe
Distance ≥ 2.0% from short strike
Example: RUT @ 2570, short put @ 2520
watch
Distance < 2.0%
Forward roll triggers here (default)
warning
Distance < 1.0%
Mirror roll triggers here (default)
critical
Distance < 0.5%
Imminent breach
breached
Price ITM (past short strike)
PUT: price ≤ strike • CALL: price ≥ strike

Calculation

# roll_service.py: _calc_breach_status()
distance_pct = (abs(price - short_strike) / price) * 100

if option_type == "PUT":
    is_breached = price <= short_strike
elif option_type == "CALL":
    is_breached = price >= short_strike

Background Scanner

When the daemon starts, it spawns an asyncio.Task that scans positions on a configurable interval.

Loop Mechanics

# app/main.py:73-91
async def _roll_scan_loop(interval: float):
    while True:
        await asyncio.sleep(interval)
        svc = get_roll_service()
        # Only during market hours: 13:30-20:00 UTC, Mon-Fri
        if now.weekday() >= 5: continue
        if hour_min < 1330 or hour_min > 2000: continue
        await svc.scan_positions()

What scan_positions() Does

  1. Gets all open multi-leg positions (credit spreads only)
  2. Fetches current market price for each symbol
  3. Calculates breach status (severity, distance%)
  4. For each position, checks:
    • Mirror: expiration day + within mirror window + severity threshold
    • Forward: severity threshold (any day)
  5. Creates RollSuggestion if eligible (deduplicates against existing pending)
  6. Expires old suggestions after TTL (5 min)
Suggestions persist in memory until executed, dismissed, or expired. Visible via GET /roll/suggestions at any time.

Historical Roll Cost Analysis

Separate analytical tool — not part of UTP. Used to study what historical rolls would have cost.

Single Ticker

    --spread-width 20 \
    --entry-breach-pcts 100 75 50 25 0 -5 \
    --target-breach-pcts 100 75 50 25 0 \
    --check-times 09:00 09:30 10:00 10:30 11:00 11:30 12:00 12:30 12:55 \
    --roll-dtes 1 2 3 5 \
    --options-dir ./options_csv_output_full_5 \
    --output-dir results_auto/roll_cost

Multi-Ticker Tabbed HTML Report

python scripts/generate_roll_cost_report.py \
    --start 2026-01-01 --end 2026-03-29 \
    --tickers RUT:20 SPX:25 NDX:50

Output CSV Columns

date, time_pst, option_type, entry_breach_pct, roll_dte, target_breach_pct,
net_roll_cost, close_debit, open_credit, price, close_short, close_long,
new_short, new_long, roll_exp
Sign convention:
  • Negative net_roll_cost = credit received (rolling pays you)
  • Positive net_roll_cost = debit paid (rolling costs you)
Limitation: each row computes ONE roll independently. Does NOT chain rolls across days. To answer "if I roll today and the position is still breached tomorrow, what's the cumulative cost?" you'd need a multi-day roll decision simulator (see Phase 2 Gaps).

Multi-Day Backtest with Rolling

The vmaxmin engine chains rolls across days but uses hardcoded rules — no "should I roll vs accept loss?" cost/benefit logic.

Run It

python scripts/backtesting/scripts/run_vmaxmin_backtest.py \
    --tickers RUT --layer --lookback-days 120 --num-contracts 40 \
    --max-rolls 99 --roll-snowball --roll-recovery-threshold 99.0 \
    --options-dir options_csv_output_full_5 \
    --max-per-transaction 500000 --max-width-multiplier 5

Roll Triggers in the Engine

TriggerCodeLogic
EOD proximity_handle_eod_rollRoll if price within proximity_threshold_pct (0.5%) of short strike
Midday breach_handle_midday_rollRoll if breach > midday_roll_breach_pct (50%)
Conditional_attempt_dte1_rollRoll only if new credit ≥ 50% of close debit

Cost Caps (Roll Halt Conditions)

The engine STOPS rolling if any of:

  • roll_count >= max_roll_count (default 3, can be 99)
  • cumulative_roll_cost >= original_credit * roll_recovery_threshold (default 0.99)

When halted, the position is closed at market.

RolledPosition State Tracked

direction, short_strike, long_strike, width, credit_per_share, num_contracts,
expiration_date,        # YYYY-MM-DD
original_entry_date,    # day the very first 0DTE was opened
original_credit,        # total $ from original 0DTE entry
cumulative_roll_cost,   # SUM of (close_debit - new_credit) across rolls
roll_count              # 1, 2, 3, ...

These objects are passed forward across days via active_carries in the multi-day backtest loop (run_vmaxmin_backtest.py:381-402).

Phase 2 Gaps

Gap 1
Live cost estimates
RollSuggestion.estimated_credit and estimated_close_cost are stubbed at 0.0. Need wiring to get_option_quotes() to compute real net_cost and covers_close.
Gap 2
Auto-execution
RollConfig.auto_execute exists but is not wired to actually fire executions automatically. Currently only scan→suggest is automatic.
Gap 3
Cost vs loss check
The service decides to suggest a roll based on breach severity alone. There's no logic that says "this roll costs more than accepting the loss — skip it."
Gap 4
Multi-day chaining
No simulator answers "if I roll today and tomorrow's position is still breached, what's the cumulative cost over N days?"

Common Workflows

Workflow 1: Live Roll Decision (UTP daemon running)

1. Daemon scans positions every 30s during market hours. Suggestions accumulate.
2. Check pending: python utp.py roll suggestions
3. Preview: curl -X POST http://localhost:8000/roll/execute/abc -H 'X-Dry-Run: true'
4. Execute: python utp.py roll execute abc --confirm
5. Or dismiss: python utp.py roll dismiss abc

Workflow 2: Manual Roll Without Waiting for Suggestion

1. View positions: python utp.py portfolio --live
2. Forward roll: python utp.py roll forward 2d9a --confirm
3. Or mirror (only on expiration day): python utp.py roll mirror 2d9a --confirm

Workflow 3: Tune Roll Aggression

Conservative: python utp.py roll config --forward-trigger breached (only roll when ITM)
Aggressive: python utp.py roll config --forward-trigger watch (roll early)
Disable mirrors: curl -X POST .../roll/config -d '{"mirror_enabled": false}'

Workflow 4: Historical "What-If" Analysis

1. Generate the full roll cost matrix:
python scripts/roll_cost_table.py --ticker RUT --start 2025-10-01 --end 2026-03-27 --spread-width 20 --output-dir results_auto/roll_cost
2. Or a tabbed multi-ticker HTML report:
python scripts/generate_roll_cost_report.py --start 2026-01-01 --end 2026-03-29
3. Open: open results/roll_cost_report/index.html

Workflow 5: Backtest a Roll Strategy

Run vmaxmin with infinite rolls + credit-neutral sizing:
python scripts/backtesting/scripts/run_vmaxmin_backtest.py --tickers RUT --layer --lookback-days 120 --num-contracts 40 --max-rolls 99 --roll-snowball --roll-recovery-threshold 99.0 --options-dir options_csv_output_full_5
